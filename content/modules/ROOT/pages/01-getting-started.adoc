== OpenShift クラスターへのログイン

まず最初に、以下の情報を使用して OpenShift クラスターにログインしてください。

*OpenShift コンソール*
link:{console_url}[{console_url},window='_blank']

*ユーザー名*

[.console-input]
[source,bash,subs="attributes+,+macros"]
----
{user}
----

*パスワード*

[.console-input]
[source,bash,subs="attributes+,+macros"]
----
{password}
----

上記の OpenShift コンソールへのリンクをクリックすると、OpenShift のログイン画面が表示されます。
ログイン画面では、入力したユーザー名とパスワードを使用してログインできます。

image::openshift-login.png[]

[#open-web-terminal]
== ワークショップWebターミナル

画面右側には、操作に必要な多くのコマンドラインツールを含むWebターミナルが表示されます。デフォルトでは、ターミナルは権限が制限されたサービスアカウントでログインされています。`{user}` としてログインするには、以下のコマンドを実行してください。

[.console-input]
[source,bash,subs="attributes+,+macros"]
----
{login_command}
----

ログイン時に、Kubernetes APIがこのワークショップでデフォルトの証明書を使用しているため、安全でない接続を使用することに関するプロンプトが表示されます。`y` と入力して承認すると、ログインが続行されます。

*ログインが完了すると、ターミナルに以下の内容が表示されます。*

[.console-output]
[source,bash,subs="attributes+,+macros"]
----
[lab-user: ~]$ {login_command}
The server uses a certificate signed by an unknown authority.
You can bypass the certificate check, but any data you send to the server could be intercepted by others.
Use insecure connections? (y/n): y

WARNING: Using insecure TLS client config. Setting this option is not supported!

Login successful.

You have access to the following projects and can switch between them with 'oc project <projectname>':

  * {user}-argocd
    {user}-bgd
    {user}-bgdh
    {user}-bgdk
    {user}-todo

Using project "user1-argocd".
Welcome! See 'oc help' to get started.
----

[TIP]
====
ターミナルにコンテンツをコピー＆ペーストするには、以下の方法があります。
セカンダリメニュー（通常は右クリックメニュー）を使用するか、
キーボードショートカットの &lt;ctrl&gt;&lt;shift&gt;&lt;v&gt; を使って貼り付けることもできます。
====
